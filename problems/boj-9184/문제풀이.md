# BOJ 9184 - 신나는 함수 실행

## 1. 문제 정보

| 항목 | 내용 |
|------|------|
| 번호 | 백준 9184 |
| 난이도 | Silver II |
| 알고리즘 분류 | 다이나믹 프로그래밍 (Memoization), 재귀 |
| 링크 | https://www.acmicpc.net/problem/9184 |

## 2. 접근 방법

### 메타 사고 과정

> **"이미 계산한 값을 왜 다시 계산해야 하지?"**

재귀 함수 `w(a, b, c)`를 순수 재귀로 실행하면, **동일한 인자 조합을 수도 없이 반복 계산**하게 된다.

예를 들어 `w(5, 5, 5)`를 계산할 때 `w(4, 4, 4)`가 몇 번 호출되는지 생각해보면, 지수적으로 폭발한다.

### 핵심 아이디어

1. 문제에서 `a > 20`이면 `w(20, 20, 20)`으로 클램핑된다.
2. 따라서 실제 계산되는 `(a, b, c)` 조합은 **`[1..20]³ = 8000가지`** 에 불과하다.
3. 한 번 계산된 결과를 3D 배열에 저장하면 → **O(1) 조회**로 재사용 가능.

이것이 **Top-Down DP (메모이제이션)** 의 전형적인 적용 패턴이다.

## 3. 알고리즘 단계

```
w(a, b, c) 호출
        │
        ▼
┌──────────────────────────────┐
│ a≤0 OR b≤0 OR c≤0 ?        │──YES──→ return 1
└──────────────────────────────┘
        │ NO
        ▼
┌──────────────────────────────┐
│ a>20 OR b>20 OR c>20 ?      │──YES──→ return w(20, 20, 20)
└──────────────────────────────┘
        │ NO
        ▼
┌──────────────────────────────┐
│ memo[a][b][c] ≠ -1 ?        │──YES──→ return memo[a][b][c]
└──────────────────────────────┘
        │ NO (아직 미계산)
        ▼
┌──────────────────────────────┐
│ a < b AND b < c ?            │
└──────────────────────────────┘
      │               │
     YES              NO
      │               │
      ▼               ▼
w(a,b,c-1)       w(a-1,b,c)
+ w(a,b-1,c-1)   + w(a-1,b-1,c)
- w(a,b-1,c)     + w(a-1,b,c-1)
                 - w(a-1,b-1,c-1)
      │               │
      └───────┬────────┘
              ▼
    memo[a][b][c] = result
              │
              ▼
          return result
```

## 4. 구현 상세

### 입력 파싱 — 종료 조건 필터링

```javascript
const problems = trimmed
    .split('\n')
    .map((str) => str.trim().split(' ').map(Number))
    .filter((p) => !(p[0] === -1 && p[1] === -1 && p[2] === -1));
```

- `-1 -1 -1` 은 입력의 종료 신호이므로, 계산 대상에서 제외해야 한다.
- 이를 빠뜨리면 결과에 `w(-1, -1, -1) = 1` 이 포함되어 오답.

### 메모 테이블 초기화

```javascript
const memo = Array.from({ length: 21 }, () =>
    Array.from({ length: 21 }, () => new Array(21).fill(-1))
);
```

- 인덱스 `[1..20]` 범위만 실제로 사용된다.
- `-1` 은 "아직 계산되지 않음"을 나타내는 sentinel 값.

### 재귀 함수 — 조건 분기 주의점

```javascript
if (a < b && b < c) {
    memo[a][b][c] = w(a, b, c - 1) + w(a, b - 1, c - 1) - w(a, b - 1, c);
} else {
    memo[a][b][c] =
        w(a - 1, b, c) + w(a - 1, b - 1, c) + w(a - 1, b, c - 1) - w(a - 1, b - 1, c - 1);
}
```

> **흔한 실수**: 조건을 `b > c` 로 쓰거나, 첫 번째 분기 수식에서 마지막 항의 부호를 `+` 로 쓰는 것.
> 반드시 문제 명세를 그대로 옮겨야 한다.

## 5. 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(20³) = O(8000) — 각 상태를 1회만 계산 |
| 공간 복잡도 | O(20³) = O(8000) — 메모 테이블 |

순수 재귀 없이 메모이제이션을 적용하면 **지수 → 선형(상태 수 기준)** 으로 폭발적으로 줄어든다.

## 6. 핵심 포인트

### 주의사항

| 번호 | 내용 |
|------|------|
| 1 | **종료 입력 필터링**: `-1 -1 -1` 은 출력하지 않아야 한다 |
| 2 | **조건 정확히**: `a < b && b < c` — `&&` 와 `<` 방향 주의 |
| 3 | **수식 부호 정확히**: 첫 번째 분기 마지막 항은 `- w(a, b-1, c)` (빼기) |
| 4 | **빈 입력 처리**: 입력이 없으면 빈 문자열 반환 |

### 메모이제이션 적용 판단 기준

- 재귀 함수가 **동일 인자로 중복 호출**되는가?  → YES
- 반환값이 **인자에만 의존**하는가(순수 함수)?  → YES
- 상태 공간이 **유한하고 충분히 작은가**?  → YES (8000가지)

이 세 조건을 만족하면 메모이제이션을 바로 적용할 수 있다.
