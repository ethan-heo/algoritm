## 2. 접근 방법

### 핵심 관찰

- N명을 두 팀으로 정확히 N/2명씩 나누어야 한다.
- 팀의 능력치는 팀원 모든 순서 있는 쌍 (i, j)에 대해 S[i][j]의 합이다.
  - 즉 i≠j인 모든 쌍: S[i][j] + S[j][i]
- 두 팀 능력치 차이의 **최솟값**을 구해야 한다.

### 메타 사고 과정

> "N명 중 N/2명을 선택하면 나머지 N/2명은 자동으로 결정된다."

N이 최대 20이므로 C(20, 10) = **184,756**가지 조합만 탐색하면 된다.
완전 탐색(백트래킹)이 충분히 동작하는 크기다.

**핵심 아이디어**: 스타트 팀 N/2명을 백트래킹으로 선택하고, 선택되지 않은 나머지를 링크 팀으로 구성하여 능력치 차이를 계산한다.

## 4. 구현 상세

### `calcScore(members)`

```javascript
function calcScore(members) {
    let score = 0;
    for (let i = 0; i < members.length; i++) {
        for (let j = 0; j < members.length; j++) {
            if (i !== j) score += S[members[i]][members[j]];
        }
    }
    return score;
}
```

- 팀원 인덱스 배열을 받아 팀 능력치를 반환한다.
- i≠j인 모든 순서쌍 (i,j)에 대해 S[i][j]를 더한다.
- S[i][j] + S[j][i] 두 방향을 모두 포함한다.

### `backtrack(idx, count, startTeam)`

```javascript
function backtrack(idx, count, startTeam) {
    if (count === n / 2) {
        // 링크 팀 = 스타트 팀에 없는 나머지
        const linkTeam = [];
        for (let i = 0; i < n; i++) {
            if (!inStart[i]) linkTeam.push(i);
        }
        const diff = Math.abs(calcScore(startTeam) - calcScore(linkTeam));
        if (diff < minDiff) minDiff = diff;
        return;
    }

    // 가지치기: 남은 인원으로 N/2명을 채울 수 없으면 종료
    const remaining = n - idx;
    const needed = n / 2 - count;
    if (remaining < needed) return;

    for (let i = idx; i < n; i++) {
        inStart[i] = true;
        startTeam.push(i);
        backtrack(i + 1, count + 1, startTeam);
        inStart[i] = false;
        startTeam.pop();
    }
}
```

- **가지치기**: 남은 인원(n - idx)이 아직 필요한 인원(n/2 - count)보다 적으면 더 진행해도 N/2명을 채울 수 없으므로 즉시 반환한다.
- `inStart` 배열로 O(1)에 스타트 팀 소속 여부를 확인한다.

## 5. 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(C(N, N/2) × N²) |
| 공간 복잡도 | O(N²) |

- N=20일 때: C(20,10) × 400 ≈ **73,902,400** 연산으로 충분히 통과 가능하다.
- 가지치기로 실제 탐색 횟수는 이보다 훨씬 적다.

## 6. 핵심 포인트

### 주의사항

1. **팀 구성의 대칭성**: 스타트 팀 {A, B}를 선택하면 링크 팀 {C, D}는 자동으로 결정된다. 0번 선수를 항상 스타트 팀에 고정하면 탐색량을 절반으로 줄일 수 있다 (`C(N-1, N/2-1)`).

2. **능력치 계산 방향**: S[i][j]는 i가 j에게 받는 능력치이므로, 팀 능력치 합산 시 (i→j)와 (j→i) 두 방향 모두 포함해야 한다.

3. **조기 종료**: 능력치 차이가 0이 되는 순간 더 탐색할 필요가 없다 (최솟값이 0이므로).

### 최적화 아이디어

```javascript
// 0번을 항상 스타트 팀에 고정 → 탐색 공간 절반 감소
// backtrack(1, 1, [0]) 으로 시작
```

이 최적화를 적용하면 C(20, 10) → C(19, 9) = 92,378로 탐색 횟수가 정확히 절반이 된다.

### 예제 검증 (예제 1, N=4)

```
S =
  0 1 2 3
  4 0 5 6
  7 1 0 2
  3 4 5 0

팀 {0,3} vs {1,2}:
  스타트: S[0][3]+S[3][0] = 3+3 = 6
  링크:   S[1][2]+S[2][1] = 5+1 = 6
  차이: |6-6| = 0 ✓
```
