# 백준 14888번 - 연산자 끼워넣기

## 1. 문제 정보

- **문제 번호**: 14888
- **난이도**: 실버 1
- **알고리즘 분류**: 백트래킹(Backtracking), 브루트포스
- **링크**: https://www.acmicpc.net/problem/14888

## 2. 접근 방법

### 메타 사고 과정

1. **문제 핵심 파악**
   - N개의 숫자와 N-1개의 연산자가 주어짐
   - 연산자의 순서를 바꿔가며 모든 경우를 계산
   - 최댓값과 최솟값을 찾아야 함

2. **제약 조건 분석**
   - 연산자 우선순위 무시 (왼쪽에서 오른쪽으로 순차 계산)
   - 나눗셈은 정수 나눗셈 (음수 나눗셈 시 `Math.trunc` 사용)
   - 연산자 개수가 제한되어 있음 (덧셈, 뺄셈, 곱셈, 나눗셉 각각의 개수)

3. **해결 전략**
   - 백트래킹으로 연산자의 모든 순열 생성
   - 각 순열마다 계산을 수행하여 결과 비교
   - 최댓값과 최솟값을 추적

### 핵심 아이디어

- **백트래킹 + 순열 생성**: 사용 가능한 연산자 개수를 추적하며 모든 조합 탐색
- **조기 종료**: 연산자를 N-1개 선택하면 계산 수행
- **상태 복구**: 재귀 호출 후 선택한 연산자를 되돌려 다른 경로 탐색

## 3. 알고리즘 단계

```
시작
  │
  ├─ 입력 파싱
  │   ├─ numbers 배열 추출
  │   └─ operatorCounts 객체 생성 {'+': a, '-': b, '*': c, '/': d}
  │
  ├─ backtrack(current, counts) 호출
  │   │
  │   ├─ [종료 조건] current.length === N-1?
  │   │   ├─ YES: 수식 계산
  │   │   │       ├─ numbers[0]부터 시작
  │   │   │       ├─ 각 연산자로 순차 계산
  │   │   │       └─ 최댓값/최솟값 갱신
  │   │   └─ NO: 계속 진행
  │   │
  │   └─ 각 연산자 순회
  │       ├─ counts[op] > 0 인가?
  │       │   ├─ YES:
  │       │   │   ├─ counts[op]--
  │       │   │   ├─ current.push(op)
  │       │   │   ├─ 재귀 호출
  │       │   │   ├─ current.pop()
  │       │   │   └─ counts[op]++
  │       │   └─ NO: 다음 연산자로
  │       └─ 반복
  │
  └─ 결과 반환 [최댓값, 최솟값]
```

## 4. 구현 상세

### 주요 함수 및 로직

#### 1) 입력 파싱
```javascript
const [, numbers, operatorList] = input
    .trim()
    .split('\n')
    .map((str) => str.trim().split(' ').map(Number));
```
- 첫 줄(N)은 무시하고 숫자 배열과 연산자 개수 배열 추출

#### 2) 연산자 카운트 객체 생성
```javascript
const operatorCounts = operatorList.reduce(
    (acc, num, idx) => {
        switch (idx) {
            case 0: acc['+'] += num; break;
            case 1: acc['-'] += num; break;
            case 2: acc['*'] += num; break;
            case 3: acc['/'] += num; break;
        }
        return acc;
    },
    { '+': 0, '-': 0, '*': 0, '/': 0 }
);
```
- 인덱스 기반으로 각 연산자의 개수를 객체로 매핑

#### 3) 백트래킹 함수
```javascript
function backtrack(current, counts) {
    // 종료 조건: 모든 연산자 선택 완료
    if (current.length === totalCount) {
        // 수식 계산 로직
        let sum = numbers[0];
        for (let i = 1; i < numbers.length; i++) {
            switch (current[i - 1]) {
                case '+': sum += numbers[i]; break;
                case '-': sum -= numbers[i]; break;
                case '*': sum *= numbers[i]; break;
                case '/': sum = Math.trunc(sum / numbers[i]); break;
            }
        }
        // 최댓값, 최솟값 갱신
        result[0] = Math.max(result[0], sum);
        result[1] = Math.min(result[1], sum);
        return;
    }

    // 각 연산자에 대해 탐색
    for (const op of operators) {
        if (counts[op] > 0) {
            counts[op]--;           // 선택
            current.push(op);
            backtrack(current, counts);  // 재귀
            current.pop();          // 복구
            counts[op]++;
        }
    }
}
```

#### 4) 나눗셈 처리
- `Math.trunc(sum / numbers[i])`: 0으로 향하는 정수 나눗셈
- 음수를 양수로 나눌 때 올림이 아닌 버림 처리

## 5. 복잡도 분석

### 시간 복잡도
- **O((N-1)! × N)**
  - 연산자 순열의 개수: (N-1)! (중복 포함 시 더 적음)
  - 각 순열마다 N개 숫자 계산: O(N)
  - 실제로는 중복 순열이므로 더 적음
  - N ≤ 11 이므로 충분히 해결 가능

### 공간 복잡도
- **O(N)**
  - 재귀 호출 스택: 최대 깊이 N-1
  - current 배열: 최대 크기 N-1
  - counts 객체: O(1) (연산자 4개 고정)

## 6. 핵심 포인트

### 주의사항

1. **나눗셈 처리**
   ```javascript
   Math.trunc(sum / numbers[i])  // ✅ 올바른 방법
   Math.floor(sum / numbers[i])  // ❌ 음수일 때 잘못된 결과
   ```
   - 음수를 양수로 나눌 때: -7 / 3 = -2 (Math.trunc), -3 (Math.floor)

2. **연산자 순서**
   - 왼쪽에서 오른쪽으로 순차 계산 (연산자 우선순위 없음)
   - `3 + 5 * 2` = `(3 + 5) * 2` = 16

3. **초기값 설정**
   ```javascript
   let result = [Number.MIN_SAFE_INTEGER, Number.MAX_SAFE_INTEGER];
   ```
   - 최댓값 초기값: 가능한 최솟값
   - 최솟값 초기값: 가능한 최댓값

### 최적화 아이디어

1. **가지치기(Pruning)**
   - 현재까지 계산한 값이 이미 최댓값/최솟값을 넘을 수 없으면 중단
   - 하지만 N이 작아서 큰 효과는 없음

2. **메모이제이션**
   - 중복 계산 방지 가능하지만, 상태 공간이 크지 않아 불필요

3. **비트마스킹**
   - 연산자 선택 상태를 비트로 표현 가능
   - 현재 구현도 충분히 효율적

## 결론

이 문제는 **백트래킹의 기본 패턴**을 익히기 좋은 문제입니다. 선택-재귀-복구의 3단계를 명확히 보여주며, 순열 생성과 완전 탐색의 조합을 활용합니다. 나눗셈 처리에 주의하면 어렵지 않게 해결할 수 있습니다.
