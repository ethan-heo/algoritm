# BOJ 24416 - 알고리즘 수업 - 피보나치 수 1

## 1. 문제 정보

- **번호**: 백준 24416
- **난이도**: Bronze 1
- **알고리즘 분류**: 수학, 재귀, 다이나믹 프로그래밍

---

## 2. 접근 방법

### 메타 사고 과정

문제의 핵심은 두 피보나치 구현 방식의 **코드 실행 횟수를 세는 것**이다.

```
재귀적 피보나치(fib1) vs. 반복적 피보나치(fib2)
```

직접 피보나치 값을 구하는 게 아니라, **각 알고리즘에서 특정 라인이 몇 번 실행되는지**를 카운트해야 한다.

### 핵심 아이디어

**fib1 (재귀)** — 기저 조건(`n == 1 || n == 2`) 도달 횟수
- 재귀 트리에서 리프 노드(base case)에 도달할 때마다 카운트
- `fib1(n)`의 base case 도달 횟수 = `fibonacci(n)` (피보나치 값 그 자체)

**fib2 (반복)** — for 루프 실행 횟수
- `for (int i = 3; i <= n; i++)` → `n - 2`번 실행
- 단, `n == 1 || n == 2`이면 루프 자체가 실행되지 않으므로 `0`번

---

## 3. 알고리즘 단계

```
입력: n

[fib1 카운트]
재귀 함수 호출
    ├─ n == 1 or 2 → count1 += 1, return 1
    └─ 그 외 → fib(n-1) + fib(n-2) 재귀

[fib2 카운트]
    ├─ n <= 2 → count2 = 0
    └─ n > 2  → count2 = n - 2

출력: count1 count2
```

### fib(5) 재귀 트리 시각화

```
                fib(5)
              /        \
          fib(4)       fib(3)
          /    \       /    \
       fib(3) fib(2) fib(2) fib(1)
       /    \   [1]    [1]    [1]
    fib(2) fib(1)
     [1]    [1]

base case 도달: 5번 → fib(5) = 5
```

---

## 4. 구현 상세

```js
function solution(input) {
    const result = [0, 0];

    // fib1: 재귀 방식
    const fibonacci = (num) => {
        if (num === 1 || num === 2) {
            result[0] += 1;   // base case 도달 횟수 카운트
            return 1;
        } else {
            result[1] += 1;   // 이 문제에서는 사용하지 않음
            return fibonacci(num - 1) + fibonacci(num - 2);
        }
    };

    fibonacci(Number(input));

    return result.join(' ');
}
```

> **주의**: 이 구현에서 `result[1]`은 재귀 분기(`else`) 진입 횟수를 세고 있으나,
> 문제에서 요구하는 fib2의 카운트는 **별도로 수식으로 계산**해야 한다.

### 정확한 fib2 카운트

문제에서 제시하는 코드 기준:

```
code 2: if (n < 3) return;
         for (i = 3; i <= n; i++) { ... }  ← 이 줄의 실행 횟수
```

| n   | fib2 실행 횟수 |
|-----|--------------|
| 1   | 0            |
| 2   | 0            |
| 3   | 1            |
| n   | n - 2        |

---

## 5. 복잡도 분석

| 구분     | fib1 (재귀)            | fib2 (반복)  |
|--------|----------------------|------------|
| 시간복잡도 | O(2^n) — 지수적 증가     | O(n)       |
| 공간복잡도 | O(n) — 재귀 콜스택 깊이   | O(1)       |

### 카운트 값 비교 예시

| n  | fib1 (base case 횟수) | fib2 (루프 횟수) |
|----|---------------------|--------------|
| 5  | 5                   | 3            |
| 10 | 55                  | 8            |
| 30 | 832040              | 28           |

---

## 6. 핵심 포인트

### 주의사항

- `fib1`의 카운트는 **피보나치 수열 값과 동일**하다. `fib(n) = fib(n-1) + fib(n-2)`의 구조에서 base case 도달 횟수가 정확히 `fib(n)`이 된다.
- `fib2`의 카운트는 단순히 `max(0, n - 2)`로 O(1)에 계산 가능하다.

### 최적화 아이디어

재귀 호출 없이 수식으로 직접 계산할 수 있다:

```js
// fib1 카운트 = fibonacci(n) 값
// fib2 카운트 = max(0, n - 2)
```

단, n이 커지면 피보나치 값은 지수적으로 증가하므로 BigInt가 필요할 수 있다.
