## 2. 접근 방법

### 메타 사고 과정

> **"길이 N의 수열을 만들 때, 마지막에 무엇이 올 수 있지?"**

`00`과 `1` 두 종류만 사용 가능하므로, 길이 N의 수열 끝에 올 수 있는 패턴은 딱 두 가지다.

- **마지막이 `1`**: 앞의 N-1 길이 수열 수 = f(N-1)
- **마지막이 `00`**: 앞의 N-2 길이 수열 수 = f(N-2)

이 두 경우는 서로 겹치지 않으므로 → **f(N) = f(N-1) + f(N-2)**

### 핵심 아이디어

작은 케이스를 직접 세어보면 피보나치 수열임을 확인할 수 있다.

| N | 수열 목록 | 경우의 수 |
|---|-----------|-----------|
| 1 | `1` | 1 |
| 2 | `00`, `11` | 2 |
| 3 | `001`, `100`, `111` | 3 |
| 4 | `0000`, `0011`, `1001`, `1100`, `1111` | 5 |

f(1)=1, f(2)=2, f(3)=3, f(4)=5, f(5)=8, ... → **피보나치 수열 (오프셋 1)**

N이 최대 1,000,000이므로 Bottom-Up DP로 O(N) 안에 해결한다.

## 3. 알고리즘 단계

```
입력 N
    │
    ▼
┌──────────────────────────┐
│ N = 1 ?                  │──YES──→ return 1
└──────────────────────────┘
    │ NO
    ▼
┌──────────────────────────┐
│ N = 2 ?                  │──YES──→ return 2
└──────────────────────────┘
    │ NO
    ▼
prev2 = 1 (f(1))
prev1 = 2 (f(2))
    │
    ▼
┌──────────────────────────┐
│ i = 3 → N               │
│   cur = (prev1 + prev2)  │
│          % 15746         │
│   prev2 = prev1          │
│   prev1 = cur            │
└──────────────────────────┘
    │
    ▼
 return prev1
```

---

## 4. 구현 상세

### 베이스 케이스 처리

```javascript
if (N === 1) return '1';
if (N === 2) return '2';
```

- f(1)=1, f(2)=2 는 반복문 시작 전에 처리
- N=1일 때 루프를 돌면 초기값이 잘못 반환될 수 있으므로 명시적 분기

### Bottom-Up DP (공간 최적화)

```javascript
let prev2 = 1; // f(1)
let prev1 = 2; // f(2)

for (let i = 3; i <= N; i++) {
    const cur = (prev1 + prev2) % MOD;
    prev2 = prev1;
    prev1 = cur;
}
```

- f(n)은 f(n-1)과 f(n-2)만 필요하므로 배열 대신 변수 2개로 충분
- 공간복잡도 O(N) → **O(1)**로 최적화
- 매 단계마다 나머지 연산을 해야 오버플로우 방지 (N=1,000,000은 15746^2 이하라 Number 범위 내)

## 5. 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(N) — i=3부터 N까지 순회 |
| 공간 복잡도 | O(1) — 변수 2개만 사용 |

## 6. 핵심 포인트

### 주의사항

| 번호 | 내용 |
|------|------|
| 1 | **나머지 연산 위치**: 덧셈 직후마다 `% 15746` 적용 — 값이 커지기 전에 나머지를 취해야 한다 |
| 2 | **베이스 케이스 분리**: N=1, N=2를 명시적으로 처리하지 않으면 루프 초기값 오류 |
| 3 | **피보나치 오프셋**: f(1)=1, f(2)=2로 일반 피보나치(1,1,2,3,5...)와 시작값이 다름 |

### 패턴 인식 방법

"경우의 수를 직접 세어봤을 때 f(N) = f(N-1) + f(N-2) 형태가 보이면" → 즉시 Bottom-Up DP 적용.
재귀 관계식이 보이는 순간 Top-Down / Bottom-Up 중 하나를 선택한다.
N이 수백만이면 Bottom-Up + 공간 최적화가 더 안전하다 (스택 오버플로우 없음).
